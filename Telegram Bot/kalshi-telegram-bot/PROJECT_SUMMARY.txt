================================================================================
PROJECT STRUCTURE
================================================================================

Prediction Flow:
  1. Scheduler triggers at 9 AM MST
  2. kalshi_client.get_playoff_markets() fetches from Kalshi API
  3. Markets saved to database with sent_date timestamp
  4. bot.send_daily_markets() formats and sends to Telegram
  5. User replies with predictions
  6. predictor_parser parses text into (team, outcome) pairs
  7. database.log_prediction() stores in predictions table
  8. Background job checks for resolved markets every 6 hours
  9. database.update_market_result() records actual outcomes
  10. User can check stats with /stats command

Database Schema:
  - predictions table: market_id, market_date, teams, prediction, outcome, timestamp, resolved, result, win_loss
  - markets table: market_id, teams, sent_date, status
  - Indexes on: market_id, market_date, resolved

CONFIGURATION OPTIONS
================================================================================

Required (must be set):
  - TELEGRAM_BOT_TOKEN      - Get from @BotFather
  - TELEGRAM_CHAT_ID        - Your Telegram chat ID
  - KALSHI_API_KEY          - From Kalshi account
  - KALSHI_API_SECRET       - From Kalshi account

Optional (have defaults):
  - KALSHI_BASE_URL         - Default: https://api.kalshi.com/v1
  - TIMEZONE                - Default: America/Phoenix (Sedona MST)
  - SEND_TIME               - Default: 09:00 (9 AM local time)
  - DB_PATH                 - Default: ./kalshi_predictions.db
  - LOG_LEVEL               - Default: INFO
  - DEBUG                   - Default: False

All configuration via .env file. No hardcoded values.
